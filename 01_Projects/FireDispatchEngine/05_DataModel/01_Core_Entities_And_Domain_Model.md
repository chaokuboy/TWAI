# 01_核心实体与领域模型

**最后更新**：2026-04-23
**标签**：#DDD #核心实体 #领域模型

## 1. 概述
本模块采用领域驱动设计（DDD）思想，定义消防接处警 7.0 系统调派引擎的核心领域实体。

## 2. 核心实体列表

| 实体名称              | 英文名                  | 职责描述                                   | 聚合根 |
|-----------------------|-------------------------|--------------------------------------------|--------|
| AlarmEvent           | 警情事件                | 记录一次警情全生命周期                     | 是     |
| EventPortrait        | 事件画像                | 存储所有槽位填充结果                       | 否     |
| AlarmLevel           | 警情等级                | 等级配置与映射规则                         | 否     |
| DispatchRule         | 调派规则                | 规则主表（等级/预案/算法）                 | 是     |
| DispatchRuleDetail   | 规则明细                | 车辆类型与数量映射                         | 否     |
| VehicleType          | 车辆类型                | 车辆字典（水罐、举高、抢险等）             | 否     |
| DispatchPlan         | 出动方案                | 最终生成的编成方案实例                     | 是     |
| FireStationVehicle   | 消防站车辆库存          | 实时车辆状态与位置                         | 否     |
| AuditLog             | 审计日志                | 全链路审计记录（不可篡改）                 | 否     |

## 3. 聚合边界说明
- **AlarmEvent** 是核心聚合根，包含 EventPortrait 和 DispatchPlan。
- **DispatchRule** 是规则聚合根，包含多条 Detail。
