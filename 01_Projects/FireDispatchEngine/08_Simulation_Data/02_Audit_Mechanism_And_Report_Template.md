# 06_审计机制与报告模板

**最后更新**：2026-04-23
**负责人**：产品经理 / 合规负责人
**标签**：#审计机制 #全链路审计 #审计报告 #不可篡改日志 #责任追溯
**适用版本**：接处警 7.0 系统调派引擎
**页面作用**：审计机制完整设计文档 + 审计报告模板

## 1. 概述

**审计机制** 是调派引擎的**事后闭环与持续优化核心**，实现全过程可追溯、可复盘、可问责，确保"数据定力"决策的透明性、合规性和持续改进。

**核心价值**：
- 满足《消防救援队伍接处警工作规范》、GB 16281-2024 等合规要求
- 支持内部审计、上级督查、战评复盘
- 90% 以上内容系统自动生成，仅人工干预点需手动补充理由

## 2. 审计核心原则

- **全覆盖**：覆盖从事件画像 → 力量计算 → 队站/车辆选取 → 约束校验 → 最终方案 → 出动反馈的全链路
- **不可篡改**：日志采用时间戳 + 哈希链式存储（区块链轻量级或数据库不可删改设计）
- **多层级访问**：基层查看本警情、中层统计分析、上级全域审计
- **自动化优先**：90% 以上审计内容系统自动生成
- **性能影响最小**：审计日志异步写入，不影响毫秒级调度

## 3. 审计内容分类与记录细节

### 3.1 输入层审计（事件画像与槽位）

- 记录内容：
  - 报警原始信息（语音转写文本、问询问题序列、报警人情绪标签）
  - 画像槽位填充详情（槽位名称、值来源、置信度）
  - 场景卡片匹配日志（匹配场景类型、理由）
- 审计价值：追溯画像准确性，识别问询引擎缺陷

### 3.2 计算层审计（力量需求与定级）

- 记录内容：
  - 完整 N_total 公式计算过程（各分项值、系数应用理由）
  - 示例日志：
    - N_base = 4（场景：高层建筑）
    - α_sub = +3（物质：锂电池）
    - β_struct = +2（高度 85m）
    - 最终 N_total = 14 → 自动定级三级警
  - 定级反向验证记录（是否上浮、理由）
- 审计价值：验证"按出动力度定级"是否严格执行

### 3.3 分配层审计（队站与车辆选取）

- 记录内容：
  - 优先级路径完整轨迹（主管队站 → ETA 补齐 → 跨区特种）
  - 每辆车实体ID、队站、出动理由
- 审计价值：检查"主管锚定 + 时空最优"平衡是否合理

### 3.4 约束校验层审计

- 记录内容：
  - 三层校验结果（人员/道路/空防）
  - 失败原因 + 回滚动作
- 审计价值：验证实战可行性保障

### 3.5 输出与反馈层审计

- 最终方案快照、出动反馈、战评结果
- 人工确认记录（理由、调整内容）

## 4. 单警情审计报告模板（系统自动生成）

**报告标题**：【警情编号】调派审计报告
**报告编号**：AUDIT-YYYYMMDD-XXXX
**生成时间**：YYYY-MM-DD HH:MM:SS

### 1. 基本信息
- 报警地址、场景类型、最终定级、N_total、出动队站数、实际到场时间、指挥员

### 2. 事件画像与输入审计（表格）
（槽位名称、填充值、来源、置信度、备注）

### 3. 计算层审计
- N_total 各分项计算过程
- 定级映射与反向验证结果

### 4. 分配层与约束校验审计
- 队站/车辆选取轨迹
- 三层约束校验结果

### 5. 人工确认记录
- 确认节点、调整内容、理由

### 6. 总结与改进建议
- 偏差分析 + 规则优化建议

## 5. 周期统计审计（周/月报）

- 警情数量、平均 N_total 偏差率
- 约束校验失败率、人工确认率
- 典型问题 Top 5 + 改进措施
